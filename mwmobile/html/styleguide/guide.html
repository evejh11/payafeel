퍼블리싱 요점
기존 운영 통합
리뉴얼을 제외한 나머지 페이지는 운영 페이지 컨텐츠만 붙임 (기존 운영 CSS 및 스크립트는 그대로 사용)
CSS 스프라이트 지원
성능 향상 목적
스크롤
주요 몇가지 기능을 제외한 좌우 스크롤은 overflow-x를 사용하여 지원 (스크립트 기능 최소화) -> 퍼블리싱팀 지원 정책
이미지 원소스
PC, Mobile 이미지를 원소스 사용해달라는 요청은 일부는 지정되었으나(현재 브랜드관), 앞으로 지원될 전반적인 내용에 대해서는 퍼블리싱팀과 협의 후 진행 필요
CSS 사용 여부 확인
vh
해당 스타일 퍼블리싱팀 확인 후 사용 결정 (4.4 지원 / 아이폰 확인 필요)
filter
vh 지원되는 기기 버전과 동일하여 사용 (4.4 지원 -webkit 추가 / 아이폰 확인 필요)
background-size
background-size에서 ‘-webkit-’ prefix 사용하지 않음 (사용하지 않아도 해당 프로젝트 기기 최저 버전 충족)
기본 스타일 가이드
컬러
리뉴얼 주 컬러는 #000, 운영페이지 공통 CSS에서 컬러를 #000로 변경 예정
폰트크기
리뉴얼 기본 사이즈는 16px이지만, 폰트 크기는 현재 운영되는 화면을 유지하기 위해서 운영 공통 CSS에서 변경할 수 없음.
renew2017.css 에서 필요한 영역에 폰트크기 재지정하여 사용
폰트 종류 (Roboto 미확정)
기본 모바일 시스템 폰트를 따라가되, 가격 또는 일부 숫자폰트는 웹폰트 (Roboto) 사용 - (적용범위는 디자인팀에서 추후 지정)
웹폰트 이슈사항 (성능, 깜빡임(?) 등) 디자인팀 전달 완료 (실제 운영단에서 보이는 것과 다를수 있음도 공지)
디자인팀 가이드
실제 가이드로 전달된 간격은 퍼블리싱과 다를 수 있음을 공지할 필요가 있음
가이드에서는 각 폰트가 가지고 있는 특징적인 여백을 고려한 것인지 확인이 필요함
그 외 아이콘은 가이드대로 사이즈를 잘를 순 있으나, 사용자의 버튼 접근성을 높여야 하기 때문에 가이드에 명시된 가이드 위치 값은 동일할 수 없음.
